# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the VTT blockchain, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, email **security@vettor.org** with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge your report within 48 hours and provide a timeline for a fix.

## Scope

This policy covers:
- VTT blockchain node (`vtt-validator`, `vtt-node`)
- Bridge smart contracts (`bridge-evm/`)
- Consensus and cryptographic primitives
- P2P networking

## Bug Bounty

We plan to launch a bug bounty program before mainnet. Details will be published on our website.
